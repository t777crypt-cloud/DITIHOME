# Столярня — сайт-каталог з адмінкою

Мінімалістичний сайт меблевого магазину. Товари додаються через сторінку
`/admin.html` і одразу з'являються у всіх відвідувачів на `/index.html`.

Структура:
```
index.html        — головна сторінка з каталогом
admin.html         — адмінка (додавання/видалення товарів)
css/style.css       — усі стилі
js/firebase-config.js — ключі підключення до Firebase (треба заповнити)
js/catalog.js       — завантаження каталогу для відвідувачів
js/admin.js         — логіка адмінки
images/placeholder.jpg — заглушка, поки немає фото
```

Сайт статичний (HTML/CSS/JS), а товари зберігаються в безкоштовній базі
даних **Firebase** — тому кожен товар, доданий через адмінку, бачать усі
відвідувачі сайту, з будь-якого пристрою.

---

## КРОК 1. Створити проєкт у Firebase (безкоштовно, ~10 хв)

1. Зайдіть на **https://console.firebase.google.com**, увійдіть через Google‑акаунт.
2. Натисніть **"Add project" / "Створити проєкт"**, назвіть його, наприклад `stolyarnya`.
3. Google Analytics можна вимкнути (не обов'язково).

### 1.1 Увімкнути Firestore (база даних товарів)
1. Ліве меню → **Build → Firestore Database**
2. **Create database** → режим **Production mode** → оберіть регіон (europe-west3 підійде) → Enable

### 1.2 Увімкнути Storage (зберігання фото)
1. Ліве меню → **Build → Storage**
2. **Get started** → Production mode → той самий регіон → Done

### 1.3 Увімкнути Authentication (вхід в адмінку)
1. Ліве меню → **Build → Authentication → Get started**
2. Вкладка **Sign-in method** → увімкніть **Email/Password**
3. Вкладка **Users → Add user** → впишіть свій email і пароль — це і буде логін в адмінку

### 1.4 Отримати ключі підключення
1. Клацніть на іконку шестерні зверху → **Project settings**
2. Прокрутіть до **"Your apps"** → натисніть значок **`</>`** (Web)
3. Дайте назву застосунку (будь-яку) → **Register app**
4. З'явиться блок `firebaseConfig = {...}` — скопіюйте ці значення
5. Відкрийте файл **`js/firebase-config.js`** у своїх файлах сайту і вставте
   свої значення замість `"ВСТАВЬТЕ_СЮДА"`

### 1.5 Налаштувати правила доступу

**Firestore → правила (Rules):**
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /products/{productId} {
      allow read: if true;
      allow write: if request.auth != null;
    }
    match /orders/{orderId} {
      allow create: if true;
      allow read, update, delete: if request.auth != null;
    }
  }
}
```
**Storage → правила (Rules):**
```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /products/{fileName} {
      allow read: if true;
      allow write: if request.auth != null;
    }
  }
}
```
Це означає: каталог видно всім, а додавати/видаляти товари може лише той,
хто увійшов через адмінку.

## КРОК 2. Викласти сайт безкоштовно (GitHub Pages)

1. Зареєструйтесь на **github.com** (якщо ще немає акаунту)
2. Натисніть **New repository**, назвіть, наприклад `stolyarnya-site`, оберіть **Public** → **Create repository**
3. **Add file → Upload files** → перетягніть туди всі файли й папки з цього проєкту (`index.html`, `admin.html`, `css/`, `js/`, `images/`) → **Commit changes**
4. Зайдіть у **Settings → Pages** → у "Source" оберіть гілку **main** → Save
5. Через 1-2 хвилини зверху з'явиться посилання виду:
   `https://ваш-логін.github.io/stolyarnya-site/`

Це і є ваш робочий сайт. Адмінка буде за адресою:
`https://ваш-логін.github.io/stolyarnya-site/admin.html`

---

## КРОК 3. Як користуватись адмінкою

1. Відкрийте `/admin.html`, увійдіть тим email/паролем, який створили в кроці 1.3
2. Заповніть форму: назва, ціна, розміри, матеріал, фото
3. Натисніть **"Додати товар"** — товар одразу з'явиться в каталозі на головній сторінці
4. Видалити товар можна кнопкою **"Видалити"** у списку внизу адмінки

## Кошик і замовлення (нове)

На сайті тепер є повноцінний кошик:
- Клієнт натискає "Додати в кошик" на товарі
- Відкриває кошик (кнопка вгорі сайту), може змінити кількість
- Натискає "Оформити замовлення" → заповнює ім'я, телефон, місто, відділення НП
- Замовлення зберігається у Firestore і одразу з'являється в адмінці, на вкладці **"Замовлення"**

**Важливо:** якщо у вас вже був старий файл `js/firebase-config.js` з правилами Firestore з попередньої версії — обов'язково оновіть правила (Rules) у Firestore, додавши блок для `orders` (див. вище, крок 1.5). Без цього кошик не зможе зберігати замовлення.

Файли, які додались/змінились: `js/cart.js` (новий), `css/style.css`, `index.html`, `admin.html`, `js/admin.js`, `js/catalog.js`. Якщо оновлюєте вже завантажений на GitHub сайт — просто перезавантажте (Upload files) усі ці файли поверх старих, GitHub сам їх замінить.

## Що ще потрібно поправити вручну (пошук і заміна в файлах)

- `js/catalog.js` → рядок `CONTACT_LINK` — вставте свій Telegram/Viber/номер телефону
- `index.html` → в footer замініть телефон, Telegram і Instagram на свої
- `index.html` → заголовок `<title>`, назву магазину "Стіл_ярня_" — можна змінити на своє

## Коли будете готові підключити оплату / домен

- Домен (напр. `stolyarnya.ua`) купується окремо в реєстратора (~200-500 грн/рік), потім прив'язується до GitHub Pages (Settings → Pages → Custom domain)
- Онлайн-оплату (LiqPay/Fondy) можна підключити пізніше, коли буде готовий ФОП —
  знадобиться API-ключ від платіжної системи, який просто вставляється в код
  кнопки замовлення

Якщо щось не запуститься з першого разу — найчастіша причина: не заповнені
ключі в `js/firebase-config.js` або не налаштовані правила доступу (Rules) з кроку 1.5.
